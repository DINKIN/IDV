The items below reflect changes since the 2.7u2 release. This version requires Java 6 or later. For most operating systems, Java 6 or higher has been the default for several years. On Mac OS-X, Java 5 is the default except for the new Snow Leopard (10.6) release. For information on upgrading Mac OS-X to Java 6, see the System Requirements. The version of netCDF-Java has been updated to 4.1. A new scheme for data caching has been instituted. Details can be found in the Performance Tuning page. A radar cross section control has been added in this release. This control allows users to create arbitrary cross sections through radar volume data. Details are in the Radar Cross Section Control section of this guide. Serveral new background image layers have been added to the Background Images control. The default layer is now a static Blue Marble image. This avoids the problem denied access problems users have been reporting becuase of the heavy use that the NASA Web Map Server gets. Several imrovements have been added to the Globe display. You can now zoom in without going through the clipping plane. You can spin the globe with a mouse flick and control the rate of rotation. There are predefined viewpoints available through the Projections menu. Most of the probes have been updated to work in the globe now. USPLN and NLDN lightning data stored in native or GEMPAK format can now be read in as point data. Several new lightning layout models have been added to the system list. Ship and buoy data are now available through the ADDE point obs chooser. A new station model has been added for these data. Starting in February, 2010, the National Weather Service will be transmitting higher resolution NEXRAD Level III radar products. This release has support for reading and displaying these new products. Many of the icons in the toolbars and display controls have been replaced with icons from the Silk Icon set from FamFamFam (http://www.famfamfam.com/lab/icons/silk/) Many bug fixes were made in this release. A few of the more obvious:
  • Suprious Map Lines (requires plugin, see Known Problems)
  • Redrawing of View Window after exiting Full Screen mode.
  • Problem with using IDV.app on Mac OS-X in 2.8
  • Wrong number of images returned from server
  • METAR plots using incorrect cloud cover parameters.
  • Handle missing data in ATCF hurricane track files
  • Handle invalid images better.
For a list of outstanding known problems, see the Known Problems page.
