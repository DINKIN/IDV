Version <%idv.version%>
The items below reflect changes since the 2.9u3 release. Upon IDV start up, memory allocation is done automatically according to the user's machine specifications, also IDV memory allocation can be changed via Edit --> Preferences, System tab. The IDV can now be installed as a Mac OS X bundle. The IDV add new user interface to add logo easily via Edit --> Preferences, View tab . A new version of the netCDF-Java library (4.2.27) is included in this release. Among the changes are a fix for Gaussian latitude grids in GRIB format. See the netCDF-Java change log for a complete list of changes. Jython library 1.5.2 is in the IDV library. A new version of ViSAD library is used in this release. Added user interface to resize the contour labels (with the consideration of the aspect ratio in the IDV), changing font, show and hide, and align (in globe view too). All wind barb displays now use the same rendering scheme. The effect is that winds less than 2.5 knots, but greater than 0 will show as a staff with no flags. Previously, they were drawn as circles (calm). Added Ensemble grids UIs to support selecting all or partial members. Added Ensemble grids diagnostic functions, including Average, Highest, Lowest, Mode, Percentile, Range, , Standard Deviation, and a single variable probability of occurrence. Added a new display type to do the Hovmoller display. Added 2D parameter as topography displays, including Color-Shaded Parameter As Topography, Contoured Parameter As Topography, Color-Filled Contoured Parameter As Topography, and Radar Sweep As Topography. Added support for GrADS dataset. Several bug fixes were made in this release, including:
  • Automatic memory allocation for Window XP 32 bit machines has been adjusted to fix a problem where some systems would not start with version 3.0.
  • the new netCDF-Java library fixed several bugs related to grids.
  • Fixed a bug when there is no animation time and the default template is being used.
  • Fixed a problem when the value in a point observation is missing (NaN).
  • Fixed a problem that was creating timeouts when accessing an OPeNDAP server.
  • Fixed a problem when generating a track image.
  • Fixed several bugs in the storm track display.
  • Fixed wind barb issue.
  • Fixed hurricane track color control issue.
  • Fixed CAP profile display bug.
For a list of outstanding known problems, see the Known Problems page.
