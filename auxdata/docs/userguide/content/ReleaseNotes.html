Integrated Data Viewer
Version <%idv.version%>
The items below reflect changes since the 2.6u2 release. The IDV now uses the non-alpha Java-netCDF 4.0 library. This will be the last version of IDV that will run under Java 5 (aka Java 1.5). With multiple core machines becoming ubiquitous, the IDV can now take advantage of using multiple processors for data reading and display. If you are running the IDV on a multi-core machine you can configure the IDV to render individual time steps in parallel. You can also do remote data reads in parallel. This typically results in a 50% reduction in overall data readings and rendering time. See the Performance Tuning page for more information. Radar RHI sweeps in DORADE and UF format can not be displayed in the Radar RHI control. This provides a view in the 3D window as well as the control window. The sounding display (e.g.SkewT) has been modified to include a 3D Hodograph and a table readout. The data in the table can be exported to a text file for import into other applications. Preliminary work has been done on reading Universal Format (UF) radar data in the IDV. We have tested on samples that we have available, but if IDV users find it does not work with their UF files, please send samples to Unidata IDV support. Users can now input a first guess field for the Barnes objective analysis of point data. When you load in point data, a new set of choices (Gridded Fields (with first guess)) is available. When you select one of these choices, you will be prompted for the grid you wish to use. The behavior of the Color Table Editor has been changed in this version. Previously, the breakpoints were overloaded to be an indication of a point and also used to paint in the edit mode. Since the painting action would change the value of the breakpoint, the new color table model uses the breakpoints only as markers. Painting is done with a dragging motion on the color bar now. For more information, see the Color Table Editor section of the User's Guide. The display settings user interface has been reworked to allow easier creation and application of Display Settings. For a list of outstanding known problems, see the Known Problems page.
