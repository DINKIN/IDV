In this section, we are going to look at the IDV's support of reading in point observation data in a text comma separated value (CSV) file format.

  1. [ht::clearDisplaysAndData]
  2. Text point data with header. 
     (index) -> (Date,Time,Latitude,Longitude,T,ROW,DIR,SPD)
  Time\[fmt="yyyy-MM-dd HH:mm:ss"\],Latitude\[unit="deg"\],Longitude\[unit="degrees west"\],T\[unit="celsius"\],
        ROW\[unit=""\],DIR\[unit="deg"\],SPD\[unit="m/s"\]
  2008-10-31, 14:00:00,36.773,98.670,9.0,43.0,320.0,1.5
  2008-10-31, 14:00:00,30.317,97.767,16.7,43.0,0.0,0.0
  2008-10-31, 14:00:00,34.398,96.148,13.0,43.0,0.0,0.0
  2008-10-31, 14:00:00,28.973,95.863,17.0,43.0,60.0,1.5
  2008-10-31, 14:00:00,45.150,89.150,8.0,43.0,310.0,2.6
           .......

    In the example above, the first line's structure is a mapping of domain parameter. The second line defines the formatting and units of the parameters. Starting the third line is the data section of this sample.

  3. Load text point data without header.
    • [ht::openDataSourceChooser]
    • In the [ht::dialog Data Source Chooser] click on the [ht::command Files] tab.
    • Go to the point data directory: [ht::filename <%workshop.datadir%>/point/].
    • Select [ht::filename metar.csv]
    • Under the [ht::command Data Source Type] pulldown menu, select "Text Point Data files"
    • Press [ht::button Add Source], and the Text Point Metadata Gui will pop up.
    • Notice the header part of this csv file being commenting out.
    [ht::cimg images/TextPointMetaDataGui.gif]
  4. Using the Text Point Metadata Gui
    • At the top a number of the initial lines from the text data are shown. The arrow keys allow you to specify the start line. For example, if you had some other header information you can skip over those commenting lines.
    • For each column of text data there is a row shown in the bottom of the dialog. This shows the sampled value and allows you to enter a name, unit, date format, missing value and extra information. These parameters are: "Time ","Latitude", "Longitude", "T", "ROW", "DIR",and "SPD".
    • After entering all the fields and you want to keep from having to do this again next time you load a new text point data of the form simply press the "Preferences" button and select [ht::menu {Save Current}]. This allows you to save these metadata settings and reapply them later using the "Preferences" button.
  5. Create the display.
    • In the [ht::command Fields] panel select [ht::param Point Data].
    • In the [ht::command Displays] select [ht::display Point Data List] and press [ht::button Create Display]
    • In the control window, use the [ht::command Layout Model] selector box to change station model from LOCATION to METAR.
